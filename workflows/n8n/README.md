# Weekly GitHub Dev Summary — n8n Workflow

**Bounty**: $200 | **Author**: sungdark | **Issue**: claude-builders-bounty#5

## Overview

An n8n workflow that automatically generates a weekly narrative summary of a GitHub repository's activity using the Claude API, and delivers it via Discord or email.

## Features

- ⏰ **Weekly cron trigger** — runs every Friday at 5pm
- 📊 **Fetches GitHub data**: commits, closed issues, merged PRs from the past week
- 🤖 **Claude API integration** — generates narrative summaries using `claude-sonnet-4-20250514`
- 🌍 **Multilingual** — supports English (EN) and French (FR)
- 📬 **Dual delivery** — Discord webhook + Email (both can be configured)
- ⚙️ **Fully configurable** via n8n variables

## Setup Instructions (5 Steps)

### Step 1: Import the workflow

1. Open your n8n instance (e.g., `https://your-n8n.com`)
2. Click **"Import from File"** → select `n8n-weekly-dev-summary.json`
3. The workflow nodes will appear

### Step 2: Configure n8n variables

Go to **Settings → Variables** and add these:

| Variable | Example Value | Description |
|----------|---------------|-------------|
| `GITHUB_REPO` | `https://api.github.com/repos/owner/repo` | Full GitHub API URL |
| `ANTHROPIC_API_KEY` | `sk-ant-...` | Your Anthropic API key |
| `DISCORD_WEBHOOK_URL` | `https://discord.com/api/webhooks/...` | Discord webhook (optional) |
| `EMAIL_TO` | `team@example.com` | Email recipient (optional) |
| `LANGUAGE` | `EN` or `FR` | Summary language |
| `LAST_WEEK_DATE` | `2026-03-20` | Auto-computed by workflow |

### Step 3: Set up HTTP Header Auth for Claude API

1. Go to **Credentials → New → Header Auth**
2. Name it `Claude API Key` (or any name)
3. Add header: `x-api-key` = `{{ $vars.ANTHROPIC_API_KEY }}`
4. In the `Call Claude API` node, select this credential

### Step 4: Connect Discord webhook (optional)

1. In Discord: Server Settings → Integrations → Webhooks → New Webhook
2. Copy the webhook URL
3. Paste into `DISCORD_WEBHOOK_URL` variable

### Step 5: Test the workflow

1. Click **"Test Workflow"** on the canvas
2. Check Discord/email for the summary
3. If successful, toggle the workflow **Active**

## Configuration Options

### GitHub Repository
- Supports any public GitHub repository
- Uses GitHub REST API v3
- Rate limit: 60 requests/hour for unauthenticated (5000 with API key)

### Language
Set `LANGUAGE` to:
- `EN` — English summary
- `FR` — French summary (français)

### Delivery Methods
Both are optional — the workflow continues if one fails:
- **Discord**: Rich embed with title, summary, and footer
- **Email**: Plain text with subject and body

## Troubleshooting

**"API rate limit exceeded"**
- Add a GitHub personal access token to the HTTP nodes as a header: `Authorization: Bearer <token>`

**Claude API errors**
- Verify your `ANTHROPIC_API_KEY` variable is set correctly
- Check your Anthropic account has available credits

**No data returned**
- Verify the `GITHUB_REPO` URL is correct (must be the API URL, not the web URL)
- Check the repo has activity in the past 7 days
