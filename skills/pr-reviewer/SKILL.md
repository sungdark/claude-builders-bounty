# SKILL.md — PR Reviewer: Structured PR Analysis Agent

## Triggers
- User says "review this PR", "/review-pr", or pastes a PR URL
- User asks "what changed in this PR?"

## What This Does
Fetches a GitHub PR diff, analyzes the changes, and returns a structured Markdown review with summary, risks, suggestions, and confidence score.

## CLI Usage
```bash
# Review a PR by URL
claude-review --pr https://github.com/owner/repo/pull/123

# Review with specific model
claude-review --pr https://github.com/owner/repo/pull/123 --model claude-sonnet-4

# Save output to file
claude-review --pr https://github.com/owner/repo/pull/123 --output review.md
```

## Script: claude-review
```bash
#!/bin/bash
set -euo pipefail

PR_URL=""
MODEL="claude-sonnet-4-20250514"
OUTPUT=""

usage() {
  cat <<EOF
Usage: claude-review --pr <PR_URL> [options]

Options:
  --pr <URL>      GitHub PR URL (required)
  --model <name>  Claude model (default: claude-sonnet-4-20250514)
  --output <file> Save output to file
  --help          Show this help

Examples:
  claude-review --pr https://github.com/torvalds/linux/pull/123
  claaude-review --pr https://github.com/vercel/next.js/pull/50000 --output review.md
EOF
}

while [[ $# -gt 0 ]]; do
  case $1 in
    --pr) PR_URL="$2"; shift 2 ;;
    --model) MODEL="$2"; shift 2 ;;
    --output) OUTPUT="$2"; shift 2 ;;
    --help) usage; exit 0 ;;
    *) echo "Unknown option: $1"; usage; exit 1 ;;
  esac
done

if [ -z "$PR_URL" ]; then
  echo "Error: --pr is required"
  usage
  exit 1
fi

# Parse owner/repo/pr_number from URL
if [[ ! "$PR_URL" =~ github\.com/([^/]+)/([^/]+)/pull/([0-9]+) ]]; then
  echo "Error: Invalid GitHub PR URL format"
  exit 1
fi

OWNER="${BASH_REMATCH[1]}"
REPO="${BASH_REMATCH[2]}"
PR_NUM="${BASH_REMATCH[3]}"

echo "🔍 Fetching PR #$PR_NUM from $OWNER/$REPO..."

# Get PR info
PR_TITLE=$(gh api "repos/$OWNER/$REPO/pulls/$PR_NUM" --jq '.title')
PR_BODY=$(gh api "repos/$OWNER/$REPO/pulls/$PR_NUM" --jq '.body // ""')
PR_AUTHOR=$(gh api "repos/$OWNER/$REPO/pulls/$PR_NUM" --jq '.user.login')
PR_CHANGED_FILES=$(gh api "repos/$OWNER/$REPO/pulls/$PR_NUM" --jq '.changed_files')

# Get diff
DIFF=$(gh api "repos/$OWNER/$REPO/pulls/$PR_NUM" --jq '.diff' @- 2>/dev/null || gh api "repos/$OWNER/$REPO/pulls/$PR_NUM" | gh api --input - --template '{{.diff}}')

echo "📊 Analyzing $PR_CHANGED_FILES changed files..."

# Build prompt for Claude
REVIEW_PROMPT="You are an expert code reviewer. Analyze this PR and produce a structured review.

## PR Info
- Title: $PR_TITLE
- Author: @$PR_AUTHOR
- Changed files: $PR_CHANGED_FILES
- PR Description: ${PR_BODY:-(none)}

## Diff
\`\`\`diff
${DIFF:0:15000}
\`\`\`

## Output Format
Produce a review in exactly this Markdown format:

### 📋 Summary
[2-3 sentence overview of what this PR does]

### ⚠️ Identified Risks
- [Risk 1 with explanation]
- [Risk 2 with explanation]
or _No significant risks identified._

### 💡 Improvement Suggestions
- [Specific, actionable suggestion 1]
- [Specific, actionable suggestion 2]
or _No major improvements suggested._

### 🎯 Confidence Score
Low | Medium | High

### 📝 Additional Notes
[Any other observations about code quality, testing, documentation]

---
*Reviewed by Claude Code PR Reviewer*"

# Call Claude API
RESPONSE=$(curl -s -X POST "https://api.anthropic.com/v1/messages" \
  -H "x-api-key: ${ANTHROPIC_API_KEY:?Please set ANTHROPIC_API_KEY}" \
  -H "anthropic-version: 2023-06-01" \
  -H "content-type: application/json" \
  -d "{
    \"model\": \"$MODEL\",
    \"max_tokens\": 2048,
    \"messages\": [{\"role\": \"user\", \"content\": $(echo "$REVIEW_PROMPT" | jq -Rs .)}]
  }" | jq -r '.content[0].text')

MARKDOWN_OUTPUT="## 🔍 PR Review: #$PR_NUM — $PR_TITLE

**Repository:** $OWNER/$REPO  
**Author:** @$PR_AUTHOR  
**URL:** $PR_URL

$RESPONSE

---
*🤖 Auto-generated by [Claude Code PR Reviewer](https://github.com/sungdark/claude-builders-bounty)*"

echo "$MARKDOWN_OUTPUT"

if [ -n "$OUTPUT" ]; then
  echo "$MARKDOWN_OUTPUT" > "$OUTPUT"
  echo ""
  echo "✅ Review saved to $OUTPUT"
fi
```

## GitHub Action: .github/workflows/pr-review.yml
```yaml
name: PR Reviewer

on:
  issue_comment:
    types: [created]

jobs:
  review:
    if: contains(github.event.comment.body, '/review')
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Parse PR URL
        id: parse
        run: |
          COMMENT="${{ github.event.comment.body }}"
          PR_URL=$(echo "$COMMENT" | grep -oE 'https://github.com/[^/]+/[^/]+/pull/[0-9]+' | head -1)
          echo "url=$PR_URL" >> $GITHUB_OUTPUT

      - name: Run Review
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          chmod +x claude-review
          ./claude-review --pr "${{ steps.parse.outputs.url }}"

      - name: Post Review Comment
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const review = fs.readFileSync('review.md', 'utf8');
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: review
            });
```

## Sample Output
```markdown
## 🔍 PR Review: #123 — Add dark mode support

**Repository:** example/app  
**Author:** @developer  
**URL:** https://github.com/example/app/pull/123

### 📋 Summary
This PR adds dark mode support via CSS custom properties and a theme toggle component. It modifies the CSS architecture to support dynamic theming without page reloads. The implementation is clean and follows existing patterns.

### ⚠️ Identified Risks
- **Flash of unstyled content**: Theme may not load before page renders, causing flash on initial load
- **LocalStorage dependency**: Theme preference stored client-side only; won't persist for users clearing browser data

### 💡 Improvement Suggestions
- Add `prefers-color-scheme` media query support for system theme detection on first visit
- Consider adding a `data-theme` attribute to `<html>` for SSR compatibility

### 🎯 Confidence Score
**High** — Well-structured PR with clear scope and good test coverage.

### 📝 Additional Notes
Tests cover the toggle functionality well. Consider adding visual regression tests for both themes.
```
