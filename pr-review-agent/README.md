# Claude Code PR Review Agent

A Claude Code sub-agent that takes a PR URL, fetches the diff, analyzes it, and posts a structured review comment.

**Bounty:** $150 — [Issue #4](https://github.com/claude-builders-bounty/claude-builders-bounty/issues/4)

---

## Features

- **CLI tool:** `claude-review --pr <url>`
- **GitHub Action:** Automatic review on every PR open/sync/reopen
- **Structured Markdown output:**
  - Summary of changes (2–3 sentences)
  - Identified risks (Critical/High/Medium/Low)
  - Improvement suggestions
  - Confidence score (High/Medium)

---

## CLI Setup

### Prerequisites

- Node.js 18+
- GitHub personal access token (`GITHUB_TOKEN` or `GH_TOKEN`)

### Installation

```bash
npm install -g .
# or
npx @claude-builders-bounty/pr-review-agent
```

### Usage

```bash
# Full URL
claude-review --pr https://github.com/owner/repo/pull/123

# Shorthand owner/repo/number
claude-review --pr owner/repo/123

# Bare number + --repo flag
claude-review --pr 123 --repo owner/repo

# Dry run (don't post comment)
DRY_RUN=1 claude-review --pr https://github.com/owner/repo/pull/123
```

### Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `GITHUB_TOKEN` | Yes | GitHub PAT with `repo` scope |
| `DRY_RUN` | No | Set to `1` to skip posting the comment |

## GitHub Action

### Option 1: Automatic review on every PR

Create `.github/workflows/pr-review.yml`:

```yaml
name: Claude PR Review
on:
  pull_request:
    types: [opened, synchronize, reopened]

jobs:
  review:
    runs-on: ubuntu-latest
    permissions:
      pull-requests: write
    steps:
      - uses: claude-builders-bounty/claude-builders-bounty/pr-review-agent@main
        with:
          github-token: ${{ secrets.GITHUB_TOKEN }}
```

### Option 2: On-demand via workflow dispatch

```yaml
on:
  workflow_dispatch:
    inputs:
      pr_url:
        description: "PR URL"
        required: true
```

### Option 3: Composite action in your repo

```yaml
- uses: claude-builders-bounty/claude-builders-bounty/pr-review-agent/.github/workflows/pr-review.yml@feature/pr-review-agent
  with:
    github-token: ${{ secrets.GITHUB_TOKEN }}
```

⚠️ Medium — manual security review recommended

## How It Works

1. Fetches PR metadata and diff via GitHub API
2. Analyzes the diff for:
   - Hardcoded secrets/credentials
   - Security-sensitive patterns (eval, exec, innerHTML, SQL mutations)
   - PR size and complexity
   - Test coverage
   - File-type-specific risks
3. Posts a structured Markdown comment on the PR
